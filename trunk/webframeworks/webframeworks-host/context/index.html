Impala Web Framework Integration Example

This page is the index page for the online demonstration of Impala's web framework support. The application consists of web modules for Struts 1, Tapestry5 and Wicket. Each of these applications is configured either using a Servlet or Filter which is declared in it's own module. Requests are intercepted and forwarded to the correct module using a single Filter/Servlet pair.

The example shows how tight integration with Impala can be achieved for arbitrary Java web frameworks, allowing web framework code as well as back end modules to be reloaded on a running JVM.

This means that you can benefit from Impala while still using your favourite web framework. You're not restricted to using Spring MVC.

In general, only minor tweaks are required to support the integration.

The Struts example demonstrates how items held in the session can be preserved across module reloads, even when the classloader used to load the object held in the session has been replaced. This allows for continuity of sessions across module reloads.

In both the Wicket and Tapestry examples, Spring beans are injected into the page. Beans can either be from the defining module, or from parent modules.

Struts - http://localhost:8080/webframeworks-web/struts/hello.do
Struts2 - http://localhost:8080/webframeworks-web/struts2/tutorial/HelloWorld.action
Wicket - http://localhost:8080/webframeworks-web/wicket/home.do
Tapestry - http://localhost:8080/webframeworks-web/tapestry5/home
JSF - http://localhost:8080/webframeworks-web/jsf/intro/intro.jsf

Modules can be reloaded using the JMX console. Also, Impala can be configured to automatically detect changes in modules and reload these, as required.
